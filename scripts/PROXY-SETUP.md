# Деплой прокси для OpenRouter (Cloudflare Workers)

Чтобы сайт не спрашивал ключ у каждого пользователя, ключ прячется на сервере-прокси.
Бесплатный тариф Cloudflare Workers даёт 100 000 запросов в день — для группы из 5-10 человек это с большим запасом.

## Шаги (3-5 минут)

### 1. Регистрация в Cloudflare

Заходишь на <https://dash.cloudflare.com/sign-up>. Email + пароль. Подтверди почту. Домен добавлять НЕ нужно — Workers работают на бесплатном субдомене `*.workers.dev`.

### 2. Создать Worker

- Левое меню → **Workers & Pages** → кнопка **Create application** → **Create Worker**.
- Имя воркера любое, например `pharma-openrouter-proxy`.
- Нажми **Deploy** (деплоится шаблон "hello world", это нормально).

### 3. Вставить код прокси

- После деплоя кнопка **Edit code**.
- Удали весь шаблонный код, вставь содержимое файла `openrouter-proxy-worker.js` (рядом с этой инструкцией).
- Кнопка **Save and deploy**.

### 4. Добавить секреты

- Вернись на страницу воркера (крестик вверху справа в редакторе).
- Вкладка **Settings** → **Variables and Secrets** → **Add variable**.

**Первая переменная:**
- Variable name: `OPENROUTER_API_KEY`
- Type: **Secret**
- Value: твой новый OpenRouter ключ (`sk-or-v1-...`)
- **Save**

**Вторая переменная:**
- Variable name: `ALLOWED_ORIGINS`
- Type: **Text**
- Value: `https://nickstr11.github.io`
- **Save**

### 5. Скопировать URL воркера

На главной странице воркера будет строка вида:

```
https://pharma-openrouter-proxy.<твой-subdomain>.workers.dev
```

Скопируй её целиком — это твой прокси.

### 6. Вставить URL в сайт

Пришли мне этот URL, я:
1. Подставлю его в `app.js` как константу `OPENROUTER_PROXY_URL`.
2. Уберу логику запроса ключа у пользователя (если прокси настроен — ключ не нужен).
3. Закоммичу и запушу.

Через минуту сайт заработает — пользователи смогут пользоваться AI-проверкой и чатом без ввода ключа.

## Безопасность

- Ключ хранится в Cloudflare secret'ах, не видно в коде воркера и тем более в исходниках сайта.
- Прокси принимает запросы только с `https://nickstr11.github.io` (проверка Origin).
- Rate limit 60 запросов в минуту на IP — от злонамеренных ботов.
- Если кто-то всё же начнёт фармить — можно либо повернуть ключ в OpenRouter, либо обновить `ALLOWED_ORIGINS` / убавить rate limit.

## Отзыв старого ключа

**Обязательно** зайди на <https://openrouter.ai/keys> и отзови старый ключ `sk-or-v1-5540...` — он светился в чате много раз. Создай новый ключ только для Worker'а, назови его `pharma-study-proxy`, установи лимит трат (например $5/месяц) чтобы в случае утечки не разориться.
